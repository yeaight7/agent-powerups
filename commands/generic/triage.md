# triage

Purpose: Turn a backlog into a clear, ranked action list.

1. Collect all open items (issues, tasks, PRs, or a provided list).
2. Read each item. Group by type: bug, feature, chore, tech debt.
3. Score by impact (high/med/low) and effort (small/medium/large).
4. Identify blockers — anything that gates other work or a release.
5. Output a ranked list: blockers first, high-impact/low-effort next, rest last.
6. Flag items that need more context before action.

Do not close, assign, or modify any items unless asked.
