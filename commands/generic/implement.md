# implement

Purpose: Test-first feature implementation that stays scoped to the spec.

1. Read the spec fully. Clarify ambiguities before starting.
2. List files to create or modify. Read them before touching them.
3. Write a failing test that captures the required behavior. Verify it fails.
4. Write the minimum code to make the test pass.
5. Run the full test suite. All tests must pass.
6. Update documentation and examples as needed.

Stay scoped to the spec. Do not commit or push unless asked.
