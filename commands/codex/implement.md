# implement

Use to turn a spec or user request into working, tested code.

Steps:

1. Read the spec or request in full. Ask if anything is ambiguous.
2. Identify the files to create or modify. Read them before editing.
3. Write a failing test for the expected behavior. Confirm it fails first.
4. Implement the minimum code that makes the test pass.
5. Run the full test suite. All tests must pass.
6. Update documentation or examples affected by the change.

Do not add features beyond the spec. Do not commit unless asked.
