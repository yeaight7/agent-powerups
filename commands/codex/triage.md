# triage

Use to process a backlog of issues or tasks into a prioritized action list.

Steps:

1. List open items from the issue tracker or provided list.
2. Read each title and description. Group by theme: bug, feature, chore, tech debt.
3. Score each item: user impact (high/med/low) and effort (small/medium/large).
4. Identify blockers — anything that prevents other work or blocks a release.
5. Output a ranked list: blockers first, then high-impact/low-effort, then the rest.
6. Flag anything that needs more information before it can be acted on.

Do not close, label, assign, or comment on issues unless explicitly asked.
