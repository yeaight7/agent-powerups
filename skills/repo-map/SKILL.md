---
name: repo-map
description: Use when the task is to understand an unfamiliar codebase, locate key entry points, or summarize architecture before editing.
---

1. Identify the project root, package manager, and main app entry points.
2. Read the smallest set of files needed to explain architecture.
3. Produce:
   - project purpose
   - entry points
   - main modules
   - data flow
   - risky areas
4. Do not edit code unless explicitly asked.
